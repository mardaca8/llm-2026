1. **Encapsulation** – the bundling of data (attributes) and methods (functions) into a single unit (a class).  

2. **Inheritance** – the OOP principle that lets a class (subclass) acquire the properties and behaviors of another class (superclass).  

3. **Polymorphism** – the ability of different objects to respond to the same method call in ways specific to their own type.  

4. **Abstraction** – hiding complex implementation details and exposing only the essential features of an object.  

5. **Constructor** – a special method that is called automatically when a new object (instance) is created to initialize its state.  

6. **Inheritance** – the “is‑a” relationship (e.g., a Dog is‑a Animal) is realized through inheritance.

8. **Private** – the access modifier that restricts a member so it can be accessed only within the class that declares it.  

9. **Method overriding** – when a subclass provides its own specific implementation of a method already defined in its parent class.

Here is a complete, ready‑to‑run Python script that does exactly what you asked for.  
It uses **pandas** to load the CSV, compute the rolling average and flag the outliers, and **matplotlib** to visualise the data.

```python
#!/usr/bin/env python3
"""
sales_analysis.py
-----------------
1. Load a CSV called 'sales.csv'.
2. Compute a 30‑day rolling average of the 'revenue' column.
3. Flag any revenue that lies more than two standard deviations away from the mean.
4. Plot the raw revenue, the rolling average, and the outliers (highlighted in red).

The script assumes the CSV has at least two columns:
    * a date/time column named 'date' (or any column you can use as the index)
    * a numeric column named 'revenue'

If the date column has a different name, just change the `parse_dates` argument
or set the index manually before running the script.
"""

import pandas as pd
import matplotlib.pyplot as plt


def main() -> None:
    # -------------------------------------------------------------------------
    # 1. Load the CSV
    # -------------------------------------------------------------------------
    # The CSV is expected in the current working directory.
    #If it lives elsewhere, adjust the path accordingly.
    try:
        # Try to parse a column called 'date' automatically.
        df = pd.read_csv("sales.csv", parse_dates=["date"])
    except FileNotFoundError:
        raise FileNotFoundError("Could not find 'sales.csv' in the current directory.")
    except ValueError:
        # If the date column is named differently (or you don't have one),
        # just read it without parse_dates – we will treat rows as sequential days.
        df = pd.read_csv("sales.csv")

    # Ensure we have a 'revenue' column.
    if "revenue" not in df.columns:
        raise ValueError("The CSV must contain a column named 'revenue'.")

    # Optional: make the date the index (handy for time‑series rolling windows)
    if "date" in df.columns:
        df = df.set_index("date")
    # If there is no explicit date column, the row number will be used as the index.

    # -------------------------------------------------------------------------
    # 2. 30‑day rolling average of 'revenue'
    # -------------------------------------------------------------------------
    # min_periods=1 ensures we get a value even when fewer than 30 rows are available.
    df["rolling_avg"] = df["revenue"].rolling(window=30, min_periods=1).mean()

    # -------------------------------------------------------------------------
    # 3. Outlier detection (>2 σ from the mean)
    # -------------------------------------------------------------------------
    mean_rev = df["revenue"].mean()
    std_rev = df["revenue"].std()
    lower_bound = mean_rev - 2 * std_rev
    upper_bound = mean_rev + 2 * std_rev

    # Flag points that lie outside the ±2σ band.
    df["outlier"] = (df["revenue"] < lower_bound) | (df["revenue"] > upper_bound)

    # -------------------------------------------------------------------------
    # 4. Plot raw data, rolling average, and outliers
    # -------------------------------------------------------------------------
    plt.figure(figsize=(12, 6))

    # Raw revenue (blue line)
    plt.plot(df.index, df["revenue"], label="Revenue", color="blue", alpha=0.6)

    # 30‑day rolling average (green line, slightly thicker)
    plt.plot(df.index, df["rolling_avg"], label="30‑day Rolling Avg",
             color="green", linewidth=2)

    # Outliers – scatter plot in red
    outliers = df[df["outlier"]]
    plt.scatter(outliers.index, outliers["revenue"],
                color="red", s=30, label="Outlier (>2σ)", zorder=5)

    # Formatting
    plt.title("Revenue and 30‑Day Rolling Average with Outliers Highlighted")
    # Use a friendly x‑label depending on the index type
    xlabel = "Date" if isinstance(df.index, pd.DatetimeIndex) else "Day Index"
    plt.xlabel(xlabel)
    plt.ylabel("Revenue")
    plt.legend()
    plt.grid(True, alpha=0.3)
    plt.tight_layout()

    # Show the plot (or save it if you prefer)
    plt.show()
    # Uncomment the following line to save the figure to a file:
    # plt.savefig("sales_analysis.png", dpi=150)


if __name__ == "__main__":
    main()
```

### How it works
| Step | What the code does |
|------|--------------------|
| **Load CSV** | `pd.read_csv` pulls the data. If you have a column called `'date'` it is parsed as a datetime; otherwise the rows are just indexed sequentially. |
| **Rolling average** | `df['revenue'].rolling(window=30, min_periods=1).mean()` computes a moving‑average smoothing over the last 30 observations. |
| **Outlier detection** | We compute the overall mean and standard deviation of the revenue, then flag any observation that lies more than **2 × std** above or below the mean. |
| **Plotting** | – Blue line = raw revenue<br>– Green line = 30‑day rolling average<br>– Red circles = points identified as outliers |
